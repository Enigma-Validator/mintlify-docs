# Liquidations

The liquidations stream uses the **multiplex WebSocket endpoint**:

```text
wss://api-eu.hypedexer.com/ws
```

All subscriptions go through `/ws`.  
You authenticate using `X-API-Key` during the WebSocket handshake.

---

## 1. Subscription schema

```json
{
  "method": "subscribe | unsubscribe | list_subscriptions",
  "subscription": {
    "type": "liquidation",
    "user": "optional user address",
    "amount_dollars": "optional minimum notional in USD"
  }
}
```

### Available filters

For `type = "liquidation"` the server currently supports:

- `user` *(optional, string)*  
  Only stream liquidations for this user address.

- `amount_dollars` *(optional, number)*  
  Only stream liquidations where `notional_usd >= amount_dollars`.

You can combine both filters.

## 3. Subscribe: Liquidations above a notional threshold

Filter by minimum notional size in USD using `amount_dollars`:

```json
{
  "method": "subscribe",
  "subscription": {
    "type": "liquidation",
    "amount_dollars": 50000
  }
}
```

Only liquidations with `notional_usd >= 50000` are streamed.

---

## 4. Subscribe: Liquidations for a specific user

```json
{
  "method": "subscribe",
  "subscription": {
    "type": "liquidation",
    "user": "0x8fcc8f55d7a8cdaee770a9e4112725d4a2c60492"
  }
}
```

You can also **combine** filters:

```json
{
  "method": "subscribe",
  "subscription": {
    "type": "liquidation",
    "user": "0x8fcc8f55d7a8cdaee770a9e4112725d4a2c60492",
    "amount_dollars": 50000
  }
}
```

---

## 5. Unsubscribe

```json
{
  "method": "unsubscribe",
  "subscription": {
    "type": "liquidation"
  }
}
```

## 6. wscat examples

### All liquidations

```bash
wscat -H "X-API-Key: <your_api_key>"   -c wss://api-eu.hypedexer.com/ws

# then:
# {"method":"subscribe","subscription":{"type":"liquidation"}}
```

### Liquidations (>= 50k USD)

```bash
wscat -H "X-API-Key: <your_api_key>"   -c wss://api-eu.hypedexer.com/ws

```
